POS 바코드 스캔 문제 — 현황 · 원인 확정 · 해결방안

일부 상품 바코드가 매장에서 스캔되지 않는 문제. 1차 원인(중량바코드 오해석)은 설정 수정으로 해소했고, 2차 원인이 본사→매장 데이터 배포 누락으로 확정되어 KPOS 담당자(박실장) 조치 요청 단계에 있습니다.

2026-07-16 개정 (초판 07-03) · 근거: HQ DB(kpos_kanzi) 판매 실측 + sang_log 교차검증 + 버우드·리드컴 실물 스캔 테스트

1경과 요약 — 원인이 두 겹이었다

설정 수정으로 증상이 바뀌면서 숨어 있던 두 번째 원인이 드러남

1차 원인 — 중량바코드 오해석 해소 (7/7)

박스히어로 자동생성 20접두 바코드 102개를 POS가 저울(중량) 바코드로 쪼개 해석 → 토탈세일즈 화면으로 점프. K_Pos Basic Setup에서 use Weight Code 해제로 해소 확인 — 증상이 "토탈세일즈 점프"에서 다음 단계 오류로 바뀜.

2차 원인 — 매장 로컬DB 미배포 확정 (7/14)

설정 해제 후 리드컴 증상: "It is not registered in the product lists" = 스캔 해석은 정상인데 매장 단말의 상품목록에 해당 품목이 없음. 같은 바코드가 버우드에서는 5종 전부 정상 스캔 → 본사 DB에는 등록돼 있으나 리드컴 단말 로컬DB에 내려가지 않은 것으로 확정.

결정적 증거: 리드컴에서 유일하게 성공한 Loop Tie는 판매 1분 전 단말에서 Edit Product 저장을 거친 상품(sang_log 기록) — 수동 터치로만 개별 유입됨.

조치 — KPOS 담당자 배포 요청 진행 중

매장 단말로의 데이터 배포(풀싱크/배치 재배포)는 KPOS 시스템 영역이라 박실장에게 조치 요청. 긴급 판매가 필요하면 해당 단말에서 상품 열고 저장하는 우회가 검증돼 있음.

3해결방안

주 경로는 KPOS 담당자 배포 조치 — 긴급 시 검증된 우회 사용

방안내용판단
① POS 설정 수정 (1차) K_Pos Basic Setup ▸ Weight Setup 탭 use Weight Code 체크 해제 → Save → 재시작중량바코드 오해석(토탈세일즈 점프) 해소. 신규 POS 설치/교체 체크리스트에 필수 등재 — 빠지면 무증상 재발 적용 완료 (리드컴)타 매장 확산 시에도 동일 적용
② 풀싱크/배치 재배포 (2차·주 경로) 박실장에게 요청: 20접두 102개 배치를 리드컴(및 전 매장) 단말 로컬DB로 재배포, 신상품이 매장에 내려가는 배포 트리거 조건 확인분석자료 일체 인계 완료 — 이 문서 + 매니저 절차서 + 사진 판독 요청·대기
③ 긴급 우회 — 단말 상품 저장 판매가 급한 상품은 해당 매장 단말에서 상품 열고(Edit Product) 저장 → 로컬DB에 개별 유입Loop Tie 사례로 실증됨 검증됨건별 수동 — 대량에는 부적합
④ 보험 — 29접두 자동발급 박스히어로 자동생성(20접두) 대신 kanzi가 양식 생성 시 발급: 29 + YYMMDD + 일련4 + 체크함정 대역(20)·KPOS 발급대역(30~32) 회피 보류②실패 시에만 실행 (7/3 전략 결정 유지)
신규 품목 등록 경로는 건강합니다 (7/15 재검증): 박스히어로 → 본사 DB 자동등록 파이프라인(kanzi)을 dry_run으로 재점검 — 품목 매핑·가격·분류 전 필드 정상. 과거 이 경로로 등록한 영숫자 바코드 상품들은 매장에서 정상 판매돼 왔으므로, 문제는 등록이 아니라 특정 배치의 매장 배포 누락에 한정됩니다.

5향후계획 — 단계별 로드맵

각 단계는 앞 단계 성공을 조건으로 진행

STEP 1 — 박실장 배포 조치 진행 중

20접두 102개 배치의 매장 재배포(풀싱크) 실행 + 신상품 배포 트리거 조건 회신. 그동안 급한 판매는 단말 상품 저장 우회로 대응.

STEP 2 — 리드컴 재검증 → 전 매장 확산 예정

재배포 후 리드컴 실패 2종 재스캔 → 성공 시 전 매장 순차 확인(각 매장 use Weight Code 설정 포함). 테스트 결제→취소로 판매 기록까지 검증.

STEP 3 — 재발 방지 예정

① POS 신규 설치/교체 체크리스트에 "use Weight Code 끄기" 필수 등재 ② 신규 품목 등록 시 "본사 등록 → 매장 배포 확인" 절차 표준화 (등록 후 매장 스캔 1건 샘플 확인).

비상 계획 — 재배포로도 해소 안 될 때

29접두 자동발급 구현(보험 보류안 실행) + 해당 102개 라벨 재발급, 그동안 상품명 검색·단말 저장 우회 병행.
