이미지 소스 확보 완료 (2026-07-10 전수조사)

박스히어로 API 전수조사(3,498품목) 결과, 1,682개(48.1%)가 이미지 보유 — 쉬즈미스 1,089 · 리스트 297 · 오조크 202 등. photo_url이 서명 없는 CloudFront 퍼블릭 URL이라 무인증 다운로드 가능 검증 완료(HTTP 200, image/jpeg 실수신). SKU 체계가 KANZI BOX와 동일하므로 매칭 → 업로드 파이프라인이 성립합니다. 나머지 51.9%는 로컬 백업 {SKU}.jpg·사방넷/신상 재수집으로 2차 보충 예정.

2개선: 엑셀 내보내기 속도·양식 컬럼 우선순위 2

데이터 관리 → 엑셀 내보내기 → 상품목록.xlsx

증상 — 다운로드가 1~2분가량 걸리고(스피너 장시간), 중간에 실패하는 경우도 있음(테스트 4회 중 2회 503). 실패 시 에러 표시 없이 스피너만 지속.

재현 결과 (2026-07-10, 4회 테스트)

시도방식결과
1, 3회차UI 버튼 클릭 (앵커 다운로드)약 35~40초 후 HTTP 503 → 파일 없음, 화면 에러 표시 없음
2, 4회차페이지 내 fetch로 끝까지 대기98초 후 200 OK — 404,835 bytes, 3,485행 × 24열 정상 파일 수신
  • 엔드포인트: GET /box/api/export/products.xlsx
  • 같은 시간대 다른 API·화면은 모두 정상 → 서버 다운이 아니라 export 생성 자체가 느림
  • 생성 완료 전(35~40초)에 중간 경로(프록시 또는 앱)가 503으로 종료 → 다운로드 실패
  • 콘솔 에러 없음 → 프론트가 503을 처리하지 않아 스피너가 무한 지속

원인 추정

3,485개 상품 × 위치 8곳 재고 조회에 98초 소요 — 이 규모(405KB)면 수 초가 정상이라, 상품별 위치재고를 개별 조회하는 N+1 쿼리 패턴(약 28,000 쿼리)일 가능성이 높습니다.

요청 사항 (속도) ① (근본) export 쿼리를 일괄 JOIN/GROUP BY로 — 수 초 내 응답 목표
② (UX) 실패 시 스피너 해제 + "잠시 후 다시 시도" 에러 메시지 표시
③ (편의) 빈 양식(헤더만) 다운로드 옵션 — 현재는 전 품목 데이터를 지우고 양식으로 사용 중

3기능: 바코드 라벨 디자이너 우선순위 3

현재: 제품명 + Code128 + SKU 고정 레이아웃, 브라우저 인쇄만 가능

필요 배경

매장 택(tag) 출력에는 가격 표시(TAG가격·판매가)와 용지 규격 대응(감열 롤·A4 라벨지)이 필수입니다. 박스히어로에서는 라벨 디자이너로 처리하던 업무로, 칸지 팀이 실제 사용 중인 템플릿은 감열지 60×40 · 50×30 · 폼텍 3100 세 가지입니다.

요청 스펙 — 1차 (핵심)

항목내용
용지 설정① 감열 라벨: 너비×높이(mm) 입력, 라벨 1장=1페이지 ② A4 라벨지: 폼텍 3100 등 규격 프리셋(칸수·여백 자동) ③ 사용자 지정 + 90° 회전 옵션
내용 편집행 단위 구성(텍스트/바코드/구분선), 순서 변경. 텍스트에 {{속성}} 변수 + 고정 문구 혼합 — 예: A${{판매가}} / {{색상}}. 기존 속성 20종 그대로 변수 사용
텍스트 스타일크기 · 굵게 · 정렬 · 최대 줄 수(말줄임)
바코드 옵션데이터 소스(SKU/바코드), 높이, 하단 텍스트 표시·크기
템플릿이름 저장 · 목록 선택 · 복제 (팀 공유). 인쇄 흐름은 기존 그대로(담기 → 수량 → 인쇄)
